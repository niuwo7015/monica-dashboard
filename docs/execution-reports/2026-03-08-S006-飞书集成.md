## 三、脚本功能说明

### feishu_notify.py

| 功能 | 说明 |
|------|------|
| 每日汇总推送 | 读取daily_tasks表，按销售分组，生成飞书卡片消息 |
| 卡片消息 | 富文本格式，含总览+各销售明细（urgent_reply/follow_up_silent/...） |
| 降级方案 | 卡片发送失败时自动降级为纯文本消息 |
| 系统告警 | `send_alert()` 函数，支持 info/warning/error 三级告警 |
| dry-run | `--dry-run` 预览消息内容，不实际发送 |
| 指定日期 | `--date 2026-03-08` 推送指定日期的任务 |
| 命令行告警 | `--alert "标题" --alert-detail "详情"` |

### feishu_sync_orders.py

| 功能 | 说明 |
|------|------|
| 认证 | tenant_access_token 方式（企业自建应用） |
| 读取表格 | 自动获取第一个sheet，也可指定FEISHU_SHEET_ID |
| 列映射 | COLUMN_MAP 配置，默认 A-G 列映射 |
| 日期解析 | 支持多种日期格式（YYYY-MM-DD、YYYY/MM/DD、Excel序列号等） |
| 金额解析 | 容错处理逗号、¥符号 |
| 销售反查 | 销售名称 → 微信号自动映射 |
| 去重 | 基于feishu_row_id（行号）upsert |
| 批量写入 | 50条/批，失败降级逐条写入 |
| 告警联动 | 同步失败/完成时自动调用 feishu_notify 发送告警 |
| dry-run | `--dry-run` 只解析不写库 |

### orders表结构

```
orders
├── id (uuid PK)
├── customer_wechat_id (text, NOT NULL, 索引)
├── sales_wechat_id (text, 索引)
├── order_date (date, NOT NULL, 索引)
├── amount (numeric 12,2)
├── product_line (text)
├── order_status (text, default 'completed')
├── feishu_row_id (text, 唯一索引)
├── remark (text)
├── synced_at (timestamptz)
├── created_at (timestamptz)
└── updated_at (timestamptz)
```
