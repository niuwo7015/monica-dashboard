## 完成内容

### 1. Dashboard.jsx 完全重写（5区块 + 时间筛选器）

**文件**: `frontend/src/pages/Dashboard.jsx`

五个数据区块：
1. **跟进覆盖率** — 环形进度图（目标70%），显示7日覆盖率 + 今日已跟/待跟/活跃客户数
2. **客户漏斗** — 加微信 → 有对话 → 已报价 → 付定金 → 成交，带层间转化率
3. **业绩** — 总金额/订单数/定金转全款率 + 各销售业绩柱状图
4. **销售跟进（今日）** — 各销售完成率进度条
5. **风险信号** — 沉默激活Top 10，显示客户名/沉默天数/最后消息

**时间筛选器**: 今天/7天/14天/30天/本月/上月/全部（影响漏斗和业绩区块）

### 2. 数据查询层

**文件**: `frontend/src/lib/dashboardQueries.js`

- 5个独立查询函数，各区块数据独立获取
- RPC优先 + fallback策略：优先调用SQL函数获取精确数据，不可用时自动降级到近似查询
- 遵守所有数据关联规则（sales_wechat_id、is_system_msg过滤、room_id群聊判断）

### 3. SQL RPC函数

**文件**: `sql/t022_dashboard_functions.sql`

3个函数 + 1个性能索引：
- `dashboard_coverage()` — 覆盖率精确计算（含chat_messages聚合）
- `dashboard_funnel_conversations(date, date)` — 漏斗"有对话"计数
- `dashboard_last_messages(text[])` — 批量获取最后消息（风险信号用）
- `idx_chat_messages_private_nonsys` — 私聊非系统消息索引

## 架构决策

| 决策 | 理由 |
|------|------|
| 查询层独立为 dashboardQueries.js | 关注点分离，方便测试和维护 |
| RPC + fallback 双模式 | SQL函数未部署时前端也能正常显示（近似数据） |
| 固定区块和时间区块分开加载 | 覆盖率/跟进/风险是固定数据（今日），不跟时间筛选器走 |
| order_stage/order_status 兼容 | 字段名不确定，代码自动检测并适配 |
