# 交接文件B：Monica销售AI系统重新设计讨论

> 目的：基于云客API对接的新能力，重新设计销售跟进系统的架构和优先级

---

## 一、核心结论（今天对话推导出的）

### 1. 销售对转化率的影响只有10-15%
- 三个销售长期转化率趋同（差2%以内）
- 客户买不买主要取决于：装修阶段、预算、产品匹配、价格竞争力
- 销售能影响的是：不丢人（覆盖率）和快慢（成交周期）

### 2. 最大杠杆是覆盖率，不是话术
- 61%的客户沉默后无跟进就流失了
- 这比优化话术的ROI高10倍
- 系统最该做的事：确保每个该跟的客户都被跟了

### 3. AI写话术的路走不通（当前阶段）
- 用HONEY案例测了3轮，每轮都不满意
- 问题不是prompt不好，是AI缺乏"此刻该推还是该松"的实时判断力
- 金牌销售的核心能力是临场读人，这个AI做不了

### 4. AI真正的战斗力是深层心理推演
- HONEY案例最终有效的分析："这个客户已经决定买了，在跟自己的钱包做心理斗争。停止解释价值，用时间紧迫性给台阶"
- 这种认知层面的洞察比话术有价值——改变销售对客户的理解方式
- 但需要extended thinking/深度推理，不是快速模式

### 5. V6诊断体系的问题
- 决策树架构（Q1/Q3/Q3_fallback）没问题
- 问题是：全在诊断（搞清客户怎么了），没有解决（然后怎么办）
- 输出应该是认知洞察+方向建议，不是具体话术

## 二、新系统四层设计

### 第一层：不漏人（规则引擎，零成本）— 最高优先
- 新客加微信未回复 → 必跟
- 客户发消息销售没回 → 必跟
- 沉默超3天 → 该跟
- 时间承诺到期（"一周后进软装"）→ 触发
- 已成交/已删除/供应商 → 排除
- 输出：每销售每天5-15人的优先级清单
- 推送到SalesToday + 飞书

### 第二层：看得懂（大模型，按需触发）
- 对清单上的客户，Opus生成3句话状态摘要
- 只在客户状态变化时跑一次
- 日均10-20次调用，成本$0.5-1

### 第三层：跟没跟（执行追踪，零成本）
- 早上推清单，晚上检查：清单上的人销售有没有发消息
- 自动生成执行率报告推给Monica
- 积累数据做学习闭环

### 第四层：深度分析（大模型，按需触发）
- 针对"跟了但卡住"的高价值客户
- 每周5-10个，Opus extended thinking深度推演
- 输出：客户真实心理状态 + 销售认知偏差 + 建议方向
- Monica/Woniu亲自看结果

## 三、云客API改变了什么

### 之前的数据管道（废弃）
手动导出xlsx → parquet → merge去重 → chat_history JSON → 诊断
问题：增量截断、语音覆盖、已删客户无标记、手动操作

### 新的数据管道
云客API自动拉取 → Supabase存储 → 规则引擎筛选 → AI诊断
- `/open/wechat/allRecords`：增量拉全部聊天记录（文本+图片+语音+视频）
- `/open/wechat/friends`：好友列表含delete字段，自动识别已删客户
- `/open/wechat/accounts`：自动获取销售微信号
- 定时任务每小时跑一次，全自动

### API已验证通过
- accounts ✅
- friends ✅（4048个好友）
- allRecords ✅（拉到31条消息验证格式正确）
- 待确认：HONEY的talker ID（调试窗口在做）

### API限制
- allRecords每次1小时数据，5秒一次
- records接口时间范围最多3天
- 只有文本和语音（records），或全类型（allRecords）
- 服务到期：2026-03-20
