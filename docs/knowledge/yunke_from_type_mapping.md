# 云客好友来源类型（fromType）对照表

> 云客API `/open/wechat/friends` 返回的好友对象中，`fromType` 字段表示该好友的添加来源渠道。
> 该字段直接透传自微信个人号内部的好友来源类型（AddScene/verifyType），非企业微信的 `add_way`。

## 1000xxx 系列 fromType 值（云客特有）

API实测发现大量 `1000000 + 基础fromType` 形式的值。推测含义为"对方主动添加"（即客户通过该渠道找到销售并发起好友申请），与基础值表示"我方主动添加"相对应。**此规律待云客官方确认。**

| fromType值 | 推测含义 | 基础值 | 实测占比 |
|-----------|---------|-------|---------|
| 1000015 | 对方通过手机号搜索添加我 | 15 | 21.8% |
| 1000003 | 对方通过微信号搜索添加我 | 3 | 2.7% |
| 1000030 | 对方扫我的二维码添加 | 30 | 2.7% |
| 1000017 | 对方通过名片分享添加我 | 17 | 0.9% |

## 重要说明

### 关于 1000xxx 系列值

这是本次调查最重要的发现。`1000015`（占总量21.8%）是除了群成员(0)以外最大的类别。推测 `1000000 + N` 表示"被动添加方向"，即：
- `fromType=15`：我方通过手机号搜索添加了对方
- `fromType=1000015`：对方通过手机号搜索添加了我方

**在系统中使用时建议：** `from_type % 1000000` 可还原为基础来源渠道，`from_type >= 1000000` 可判断是否为对方主动添加。

### 关于 fromType=0

占比最高（32.7%），且 **全部为 type=2（群成员）**。确认 `0` 表示"未知来源"，在好友列表中主要出现在群成员记录（非好友关系）。

### 关于 fromType=6

值 `6` 在 wechatsdk.com 文档中被定义为"好友验证消息（默认值）"，而非"扫描二维码"。占比4.5%。当微信无法明确识别具体添加方式时（如对方已有你的微信号、历史迁移联系人等），会使用此默认值。

### 关于 fromType=81

占比5.5%，在所有公开文档中均未找到定义。根据微信功能特性和数值范围推测，可能对应"朋友圈广告"或"微信广告"渠道。**仅出现在可欣和晴天喵的账号中**，这两位可能承接了更多广告渠道来的客户。

### 关于 fromType=30

占比0.9%，API实测已确认存在。推断为"扫一扫二维码"添加，与 `1000030`（对方扫我的码）相对应。

### 企业微信 add_way 对照（注意：编号体系不同）

企业微信的 `add_way` 是另一套编号体系，不可与个人微信 `fromType` 混用：

| add_way | 含义 |
|---------|------|
| 0 | 未知来源 |
| 1 | 扫描二维码 |
| 2 | 搜索手机号 |
| 3 | 名片分享 |
| 4 | 群聊 |
| 5 | 手机通讯录 |
| 6 | 微信联系人 |
| 7 | 来自微信的添加好友申请 |
| 8 | 安装第三方应用时自动添加的客服人员 |
| 9 | 搜索邮箱 |
| 201 | 内部成员共享 |
| 202 | 管理员/负责人分配 |
